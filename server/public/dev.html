Warboard Developer API v1 · Preview

Public API for building third-party tools on Warboard. If you're making a Torn Discord bot, mobile app, analytics dashboard, or anything else that wants faction data, start here.

Base URL: https://tornwar.com · OpenAPI: /api/v1/openapi.json

Preview status. The v1 API is versioned and stable within the v1 line, but endpoints may be added. Breaking changes would ship as v2.

Quick start

1. Get a Personal Access Token

Exchange your Torn API key for a long-lived token. The token is shown once — save it.

curl -X POST https://tornwar.com/api/dev/pat \
  -H "Content-Type: application/json" \
  -d '{
    "key":    "YOUR_TORN_API_KEY",
    "scopes": ["read:oc", "read:faction", "manage:webhooks"],
    "name":   "my-discord-bot"
  }'

Response:

{
  "token": "wb_pat_abcdef...",   // save this
  "pat": { "id": "pat_123...", "scopes": [...], ... }
}

2. Call a v1 endpoint

curl https://tornwar.com/api/v1/oc/spawn?factionId=42055 \
  -H "Authorization: Bearer wb_pat_abcdef..."

3. (Optional) Register a webhook

curl -X POST https://tornwar.com/api/webhooks \
  -H "Authorization: Bearer wb_pat_abcdef..." \
  -H "Content-Type: application/json" \
  -d '{
    "url":    "https://myapp.example.com/warboard-webhook",
    "events": ["vault-request.*", "chain.milestone", "war.started"]
  }'

Response contains secret — used to verify X-Warboard-Signature on incoming POSTs.

Authentication

All v1 endpoints require:

Authorization: Bearer wb_pat_<your_token>

Rate limit: 100 requests/minute per PAT.

Webhook delivery format

When an event fires for a subscribed webhook, Warboard POSTs JSON:

POST /your-webhook-url
Content-Type: application/json
X-Warboard-Signature: sha256=<hmac-hex>
X-Warboard-Event: vault-request.created
X-Warboard-Event-Id: evt_abcdef...

{
  "id": "evt_abcdef...",
  "type": "vault-request.created",
  "factionId": "42055",
  "occurredAt": 1745...,
  "data": {
    "requestId": "vr_xyz...",
    "requesterId": "137558",
    "requesterName": "RussianRob",
    "amount": 500000
  }
}

Verify with:

const crypto = require("node:crypto");
const expected = "sha256=" + crypto.createHmac("sha256", WEBHOOK_SECRET)
  .update(rawBody).digest("hex");
if (expected !== req.headers["x-warboard-signature"]) reject();

Subscribe to wildcards (vault-request.*) or all events (*).

Retries & failure handling

  • At-least-once delivery; dedupe by X-Warboard-Event-Id.
  • Retries: 2s → 8s → 32s backoff on 5xx / network error.
  • After 10 consecutive delivery failures, the webhook auto-disables. Re-enable via API.
  • Delivery timeout: 10 seconds.
