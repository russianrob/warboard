Terms of Service & Privacy

Last updated: March 16, 2026

No API Keys Stored on Server

Your Torn API key is stored exclusively in your browser's local storage. It is never transmitted to, stored on, or accessible by the FactionOps server. The server never sees, logs, or retains your personal API key at any point.

No Personal Data Stored

FactionOps does not store, log, or retain any personal information on the server. No user profiles, no browsing history, no IP addresses, no analytics, no cookies. The server holds only ephemeral war coordination data (calls, priorities) that is discarded when the war ends.

1. What FactionOps Is

FactionOps is a free userscript and companion server for Torn.com that provides real-time faction war coordination. It runs as a browser extension via Tampermonkey or Violentmonkey and communicates with a lightweight coordination server over WebSockets.

2. Data We Do Not Collect

FactionOps does not collect, store, transmit, or have access to any of the following:

  • Torn API keys — Your API key never leaves your browser. It is stored in your browser's local storage and is used exclusively by the userscript to make direct requests to the official Torn API from your browser. The FactionOps server has no mechanism to receive, read, or store your API key.
  • Personal information — No real names, email addresses, IP addresses, browser fingerprints, device identifiers, or any other personally identifiable information.
  • Browsing activity — No page history, click tracking, session recordings, or behavioral analytics of any kind.
  • Game data for profit — No scraped game data is sold, shared with third parties, or used for any purpose beyond real-time war coordination within your faction.
  • Cookies or tracking — FactionOps sets no cookies, uses no third-party analytics services, and does not track users across sessions.

3. Data That Passes Through the Server

The FactionOps server acts purely as a real-time relay for faction coordination. The following ephemeral data is transmitted between connected faction members during active wars:

  • Target calls — Which enemy targets have been called by which faction member (Torn player ID and name only).
  • Priority tags — Target priority levels set by faction leaders (high, medium, low).
  • Viewing indicators — Which faction members are currently viewing the same target (for group attack detection).
  • Connection status — Whether a faction member's script is currently connected (online/offline indicator).

This data is held in volatile memory only. It is not written to any database, not logged to disk, and is discarded when the server restarts or the war ends. There is no historical record, no audit trail, and no data retention.

4. Faction API Key (Optional)

Faction leaders may optionally provide a dedicated Limited Torn API key for server-side status polling. This is a separate key from your personal API key, created specifically for FactionOps. If provided:

  • It is used solely to poll the Torn API for faction member statuses during active wars.
  • It is stored on the server in an encrypted configuration file, accessible only to the server process.
  • It can be removed at any time by the faction leader through the settings panel.
  • It should be a Limited access key to restrict its permissions to the minimum required.

This is entirely optional. FactionOps functions without it — the local userscript will continue to update statuses from the page you are actively viewing.

5. How the Userscript Works

The FactionOps userscript operates exclusively on Torn.com war and faction pages. It:

  • Reads war member data from pages you are actively viewing to update the local UI (sorting, timers, status colors).
  • Communicates with the FactionOps coordination server to sync calls, priorities, and viewer indicators with your faction members.
  • Makes requests only to the official Torn API (api.torn.com) and the FactionOps server — no other external services.
  • Does not make any non-API requests to Torn.com that are not directly initiated by you.
  • Does not intercept, bypass, or interfere with any CAPTCHA protections.
  • Does not operate on background tabs, minimized windows, or unfocused pages.

6. Torn Scripting Compliance

FactionOps is designed to comply with Torn's official scripting and scraping rules as published at torn.com/rules.php and the Updated Rules Page: Scripting & Scraping forum thread. Specifically:

  • The userscript only processes data from pages you are actively viewing and interacting with.
  • Server-side data comes exclusively from the official Torn API using authorized API keys.
  • No page scraping data is forwarded to external systems.
  • All functionality is fully disclosed.
